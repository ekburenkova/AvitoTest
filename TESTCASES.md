### Tест-кейс №1. Создание объявления с валидными данными

Метод: POST 

Ресурс: /api/1/item

Тип: Позитивный

Приоритет: Высокий

Шаги:

1. Отправить POST запрос с валидными данными

Ожидаемый результат:

* Статус-код 200
* В ответе есть поле status
* В поле status значение содержащее текст "Сохранили объявление"
* В поле status есть идентификатор объявления в формате UUID

### Tест-кейс №2. Создание объявления без обязательного поля

Метод: POST

Ресурс: /api/1/item

Тип: Негативный

Приоритет: Высокий

Шаги:

1. Отправить POST запрос без поля name

Ожидаемый результат:

* Статус-код 400
* В ответе есть поле message (внутри поля result)
* В поле message текст "поле name обязательно"


### Tест-кейс №3. Идемпотентность создания объявления

Метод: POST

Ресурс: /api/1/item

Тип: Корнер-кейс (идемпотентность)

Шаги:

1. Отправить одинаковый POST запрос (с валидными данными) дважды

Ожидаемый результат:

* Статус-код 200
* Создаются два объявления с разными id


### Tест-кейс №4. Получение объявления по валидному id (e2e)

Метод: GET

Ресурс: /api/1/item/{id}

Тип: Позитивный

Шаги:

1. Отправить POST запрос для создания объявления с валидными данными
2. Отправить GET запрос с существующим id

Ожидаемый результат:

* Статус-код 200
* Значение поля id соответствует переданному в GET id
* Значения name и price соответствуют переданным


### Tест-кейс №5. Получение объявления по id в формате uuid

Метод: GET

Ресурс: /api/1/item/{id}

Тип: Негативный

Шаги:

1. Отправить GET запрос с несуществующим (рандомным) id

### Tест-кейс №6. Получение объявлений продавца (e2e)

Метод: GET

Ресурс: /api/1/{sellerId}/item

Тип: Позитивный

Шаги:

1. Создать несколько (2) объявлений с одним sellerId с помощью POST
2. Отправить GET запрос

Ожидаемый результат:

* Статус-код 200
* Возвращается список объявлений с одинаковым sellerId


### Tест-кейс №7. Получение статистики по объявлению (e2e)

Метод: GET

Ресурс: /api/1/statistic/{id}

Тип: Позитивный

Шаги:

1. Создать объявление через метод POST
2. Запросить статистику через метод GET

Ожидаемый результат:

* Статус-код 200
* В ответе есть поля статистики ("contacts", "likes", "viewCount")
* Значения всех полей статитстики имеют числовой (int) тип

### Tест-кейс №8. Проверка времени ответа API при получении статистики (e2e)

Метод: GET

Ресурс: /api/1/statistic/{id}

Тип: Нефункциональный

Шаги:

1. Создать объявление через POST
2. Отправить GET запрос на получение статистики
3. Замерить время ответа

Ожидаемый результат:

* Статус-код 200
* Время ответа не превышает 1 секунды
